# 1. 支付方式

## 1.1 支持的支付方式
- App：支持第三方支付、支付卡、礼品码组合支付。
- 小程序：支持第三方支付、礼品码支付。
- 官网：支持第三方支付、支付卡、礼品码组合支付。

## 1.2 组合支付规则
- 同一订单最多可使用3张礼品码。
- 如果礼品码金额不足，可与1种在线支付方式组合使用。
- 同一订单不可同时使用2种在线支付方式。

## 1.3 支付时限
- 实物订单提交后15分钟内未完成支付，则订单自动关闭。
- 预售商品若商品页另有支付时限说明，则以商品页说明为准。
- 订单关闭后，库存和优惠资格不保留。

# 2. 支付失败

## 2.1 常见原因
- 支付页面停留时间过长。
- 网络中断。
- 风控校验未通过。
- 礼品码失效或余额不足。
- 支付渠道处理中，订单状态未及时回写。

## 2.2 处理规则
- 如果支付页面提示失败，用户可返回订单列表查看最终状态，不建议重复提交多个新订单。
- 如果用户显示扣款成功但订单仍为“待支付”，应等待30分钟系统回写。
- 如果等待30分钟后仍未同步，则客服应创建支付核查工单。
- 如果同一用户重复支付多笔相同订单，应先核对是否生成多张订单，再分别处理。

# 3. 退款原路退回

## 3.1 退款路径
- 所有退款默认按原支付路径退回。
- 如果订单使用礼品码支付，则礼品码退款部分原路退回礼品码账户。
- 如果订单使用组合支付，则按原支付构成分别退回。

## 3.2 到账时效
- 礼品码退款：通常实时至24小时内。
- 第三方支付退款：通常1至5个工作日。
- 支付卡退款：通常3至7个工作日。
- 如果超过预计时效仍未到账，应创建退款跟踪工单。

## 3.3 退款失败
- 如果原支付路径异常导致退款失败，系统会自动重试一次。
- 如果重试后仍失败，则转人工核实处理方式。
- 客服不得口头承诺更改为任意其他账户收款。

# 4. 电子发票

## 4.1 发票类型
- 仅支持电子发票。
- 个人订单支持电子普通发票。
- 企业订单支持电子普通发票。
- 不支持纸质发票。
- 不支持专用发票。

## 4.2 开票时点
- 实物订单需在订单状态变为“已完成”后申请开票。
- 会员权益类虚拟商品在权益到账后可申请开票，前提是未发生退款。
- 一个订单仅支持开具一次发票。

## 4.3 开票金额
- 发票金额以商品实付金额为准。
- 运费、优惠券抵扣、积分抵扣部分不计入开票金额。
- 如果订单部分退款，发票金额需按实际保留成交金额调整。

## 4.4 申请期限
- 发票申请期限为订单完成后180个自然日内。
- 超过180个自然日未申请，则系统不再支持自助申请，需转人工复核，最终不保证可补开。

# 5. 发票修改

## 5.1 未开票前修改
- 如果订单尚未开票，则用户可修改发票抬头信息后再提交申请。
- 同一订单未开票前可修改不限次数，但以最终提交信息为准。

## 5.2 已开票后修改
- 如果电子发票已开具，原则上不支持直接修改。
- 如果用户确有修改需求，需同时满足：
  1. 发票开具后30个自然日内提出；
  2. 发票未被使用；
  3. 原发票可作废。
- 满足条件后，可申请原票作废并重开一次。

## 5.3 退款与发票联动
- 如果订单已开票后发生全额退款，原发票需作废。
- 如果订单已开票后发生部分退款，则原发票可能作废并按剩余成交金额重开。
- 客服不得承诺“退款了发票一定不用处理”，应以系统联动结果为准。

# 6. 企业发票限制

## 6.1 抬头要求
- 企业发票需填写企业全称和纳税人识别号。
- 企业名称与纳税人识别号不一致时，系统可能拒绝开票申请。
- 一个订单仅支持一个企业抬头，不支持拆分到多个抬头。

## 6.2 不可开企业发票的情形
- 礼品赠送活动赠品部分不可单独开票。
- 优惠券抵扣、积分抵扣部分不可开票。
- 同一订单中的运费不可单独拆开为单独发票。
- 已全额退款订单不可新开发票。

## 6.3 复核场景
- 如果企业抬头信息多次修改、税号格式异常、申请超期，则需转人工复核。
- 人工复核通过前，客服不得承诺“今天一定开出”。
