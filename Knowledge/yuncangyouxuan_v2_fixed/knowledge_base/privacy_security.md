# 1. 信息脱敏规则

## 1.1 手机号脱敏
- 客服界面默认仅展示前3位和后4位。
- 展示格式示例：138****5678。
- 如果需要核验身份，客服仅可让用户确认后4位，不可要求报完整号码。

## 1.2 身份证号脱敏
- 客服界面默认仅展示前3位和后4位。
- 展示格式示例：110***********1234。
- 普通在线客服不得查看完整身份证号。

## 1.3 支付卡号脱敏
- 客服界面默认仅展示前6位和后4位。
- 展示格式示例：621234******5678。
- 客服不得在聊天中要求用户发送完整支付卡号。

## 1.4 地址脱敏
- 客服界面仅展示省、市、区县和部分街道信息。
- 详细门牌号、楼层、房号默认脱敏展示。
- 如果需要核验订单地址，客服仅可请用户确认区县和门牌号后2位，不可整段复述完整地址。

# 2. 客服不能索要的信息

- 不能索要登录密码。
- 不能索要短信验证码。
- 不能索要支付密码。
- 不能索要完整身份证号照片正反面。
- 不能索要完整支付卡正反面照片。
- 不能索要含二维码的付款截图原图。
- 不能要求用户通过聊天窗口发送手持证件照。
- 如果确需身份复核，只能引导至官方安全验证入口或转人工安全专员处理。

# 3. 账户异常处理

## 3.1 异常表现
- 用户反馈账号被登出、无法登录、地址被修改、优惠券异常消失、出现非本人订单。
- 系统提示风险拦截、设备异常、临时冻结。

## 3.2 处理步骤
- 第一步：确认用户账号标识，如手机号后4位、注册渠道、最近一笔订单号。
- 第二步：判断是否涉及资金类风险、账号被盗或信息泄露风险。
- 第三步：如涉及高风险，立即建议用户修改密码并退出其他设备。
- 第四步：创建账户安全工单并转人工复核。

## 3.3 临时限制规则
- 如果系统判定存在异常登录风险，账号可能被限制30分钟至24小时的部分功能。
- 限制期间，用户可能无法下单、修改地址、申请高价值退款。
- 客服不得绕过系统直接解除限制。

# 4. 支付异常处理

## 4.1 支付成功但订单未更新
- 如果用户反馈已支付但订单仍显示待支付，应先建议等待30分钟。
- 如果30分钟后仍未更新，则创建支付核查工单。
- 在核查完成前，不得让用户重复提交敏感支付信息。

## 4.2 重复支付疑似
- 如果用户反馈同一商品支付了两次，客服应先核对是否生成两张不同订单。
- 如果是同一订单重复支付，以支付核查结果为准处理退款。
- 如果是两张独立订单，需分别判断是否发货、是否可取消。

## 4.3 风险提醒
- 客服不得建议用户将款项转到任何个人账户处理。
- 客服不得发送外部收款二维码。
- 所有补款、退款、核验均应在官方订单与官方页面内完成。

# 5. 转人工规则

## 5.1 必须转人工的情形
- 用户明确表示账号被盗或存在非本人下单。
- 用户要求修改已实名信息且系统无法自助处理。
- 用户涉及高价值订单的支付异常、退款异常、签收争议。
- 用户要求提供隐私数据访问记录或删除申请。
- 用户投诉客服索要了敏感信息。

## 5.2 优先转人工的情形
- 用户连续2次安全校验失败。
- 用户无法通过基础信息完成身份核验。
- 用户表示正在遭遇诈骗、冒充客服联系等风险。

## 5.3 转人工前话术要求
- 应先告知“为保障账户和隐私安全，该问题需要由人工安全专员进一步核验”。
- 不得使用“这个我看不到，你自己处理”类表述。
- 不得承诺人工一定立即解除限制，应说明“以安全复核结果为准”。

# 6. 数据最小化原则

- 能用后4位核验的信息，不要求完整信息。
- 能用订单号核验的问题，不额外索要证件信息。
- 能通过系统记录确认的问题，不要求用户重复提交敏感材料。
- 客服记录工单时，应只记录解决问题所需的最小必要信息。
