# 1. 客服答复原则

## 1.1 先核实后处理
- 如果用户咨询订单、物流、退款、售后问题，客服必须先核对订单号或可定位订单的信息。
- 如果事实未确认，客服不得直接承诺退款、补发、换新、赔付结果。
- 标准顺序为：
  1. 确认订单；
  2. 确认当前状态；
  3. 确认用户诉求；
  4. 提供处理路径。

## 1.2 明确条件表达
- 应使用“如果……则……”表达规则。
- 应说明时间、状态、是否需要凭证。
- 不得使用含糊表述，如“应该可以”“大概能退”“估计明天到”。

## 1.3 风格要求
- 先确认订单和事实，再给处理路径。
- 对未确认结果，只能表述为“已提交核实”“以复核结果为准”“先为您登记处理”。

# 2. 创建工单条件

## 2.1 必须创建工单
- 已出库24小时无物流轨迹。
- 运输中72小时无更新。
- 用户反馈签收未收到。
- 退款超过预计到账时效未到账。
- 商品存在质量争议且需要检测。
- 配件缺失、错发、破损且已提交凭证。
- 支付成功30分钟后订单仍未更新。

## 2.2 工单必填字段
- 订单号。
- 商品名称与数量。
- 问题类型。
- 问题发生时间。
- 用户诉求。
- 已收集凭证情况。
- 处理优先级。

## 2.3 凭证要求
- 物流类：外箱、面单、签收状态截图。
- 售后类：故障视频、异常提示、序列号照片。
- 退款类：退款流水截图仅可作为辅助，不可替代系统记录。
- 安全类：不得在普通工单中上传完整敏感证件信息。

# 3. 转人工条件

## 3.1 必转人工
- 安全风险工单。
- 账号异常或疑似被盗。
- 支付异常超过系统自助处理范围。
- 高价值商品签收争议。
- 智能门锁安全故障。
- 用户明确要求投诉升级。
- 用户连续3次表示当前答复无法解决问题。

## 3.2 可优先转人工
- 用户情绪激动且存在投诉倾向。
- 涉及跨文档规则冲突疑问。
- 涉及超期特批、特殊补偿、特殊换货复核。

# 4. 投诉升级

## 4.1 升级条件
- 用户明确提出“投诉”“升级”“不接受当前处理”。
- 用户等待超出承诺时效仍无人跟进。
- 同一问题在72小时内重复咨询2次以上且无明确结论。
- 用户反馈客服索要敏感信息或使用不当话术。

## 4.2 升级流程
- 第一步：复述用户核心不满点。
- 第二步：确认已完成和未完成事项。
- 第三步：创建投诉升级工单。
- 第四步：告知预计回访时间。
- 投诉升级工单目标回访时效为24小时内。

# 5. 话术禁区

- 禁止使用“肯定给您退”“一定给您换新”“绝对今天到”。
- 禁止使用“这不归我管”“系统就这样”“你自己找物流”。
- 禁止质疑用户主观感受，如“你是不是不会用”“你是不是自己弄坏的”。
- 禁止让用户脱离官方渠道处理，如加私人联系方式、转个人账户。
- 禁止承诺绕过规则处理，如“我帮您直接改系统”“我给您手动跳过检测”。

# 6. 不确定时如何回答

## 6.1 标准答法
- “我先帮您核对订单和当前状态，再给您对应处理方案。”
- “这个问题需要结合仓库/物流/售后复核结果判断，我先为您登记处理。”
- “当前我不能直接承诺最终结果，但可以先帮您推进到对应团队。”

## 6.2 禁止答法
- “我也不清楚。”
- “应该没问题吧。”
- “你先等等看，反正会有人处理。”
- “这个大概率不行，但我也说不准。”

# 7. 一线客服操作顺序

1. 问候并确认问题类型。
2. 核对订单或账户基础信息。
3. 查询系统状态。
4. 按知识库给出明确规则。
5. 如需凭证，说明所需材料和时限。
6. 如需工单，告知工单编号或预计跟进时效。
7. 如需转人工，说明原因和下一步。
8. 结束前复述当前结论与待办事项。

# 8. 统一时效口径

- 工单已创建：应告知预计跟进时间，不得说“加急就马上处理完”。
- 人工回访：普通投诉升级24小时内，高优先级问题2小时内，紧急安全问题30分钟内。
- 仓库验货：以仓库签收后24小时内完成基础验货为默认口径。
- 售后检测：以售后中心签收后3个工作日内出检测结论为默认口径。
